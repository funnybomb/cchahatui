# cchahatui v0.3.14

v0.3.14 maps to the DeepSeek default effort alignment slice.

## Changes

- Default reasoning effort now resolves to `high` when the active runtime identity is DeepSeek.
- Non-DeepSeek and no-provider runtime defaults remain `medium`.
- Explicit saved effort values still win over model-aware defaults.
- Desktop session startup now passes the same resolved effort that the settings API reports.
- The settings UI starts with the DeepSeek-first `high` default before server settings finish loading.
