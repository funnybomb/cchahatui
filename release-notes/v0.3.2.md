# cchahatui v0.3.2

v0.3.2 is the project workspace release: it aligns the desktop layout with the current Codex-style workbench, adds project navigation, introduces project memory, and improves web/H5 action parity.

## Features

- Refined the desktop layout and typography baseline with a wider project sidebar, tighter chat column, compact session header, and cleaner workspace panel surface.
- Added project navigation groups in the left sidebar, including pinned projects, empty saved projects, missing-directory state, and project-level actions.
- Added a keyboard shortcut help dialog opened from the sidebar or `Cmd/Ctrl+/`.
- Added structured project memory with summary, facts, decisions, open tasks, update metadata, and a per-project reuse toggle.
- Reused enabled project memory automatically across new chats and follow-up messages in the same project without changing visible user text.
- Added explicit project and session action buttons so web/H5 users can access the same menus without depending on native right-click.

## Fixes

- Fixed long workspace preview tests under coverage by giving large preview expansion cases enough timeout headroom.
- Fixed changed-line coverage so non-executable desktop CSS is excluded from V8 executable-line coverage evaluation.
- Fixed a browser startup race where desktop dev mode could temporarily point API calls at the Vite frontend port before the local server was verified.
- Fixed Agent Team display names stored in sanitized directories, so names like `My Team` now resolve correctly for team details, member transcripts, member messages, and delete operations.
- Fixed member session tabs so transcript polling restarts when a member tab becomes active again.
- Fixed desktop team watcher handling when update or delete events use sanitized team directory names.
- Fixed team deletion cleanup so synthetic member tabs are closed and focus returns to the leader tab when available.
